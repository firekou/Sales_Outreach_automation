# Lead 分析 + 三封信輸出模板

每筆 HOT/WARM Lead 的標準產出。複製此模板填寫；完成後可存到 `lead-drafts/lead-00x-<name>.md`，
並把 custom1/2/3 帶入 Dripify/Expandi CSV。對齊 `AirTalk-S1-BizDev` agent 的輸出格式。

---

```markdown
# Lead 0XX — [姓名]

**建立日期：** YYYY-MM-DD ｜ **策略：** S1 直攻法 ｜ **發送帳號：** [frank/jet/...]

## 基本資料
| 欄位 | 內容 |
|------|------|
| 姓名 | |
| 職稱 | |
| 公司 / 規模 | |
| LinkedIn URL | |
| 搜尋組合 | [A/B/C/D/E/G] |
| ICP 分數 / 分類 | [分數] / [HOT/WARM/COLD] |

## 【分析】
- **AI 使用者類型**：[A/B/C/D]（[類型名稱]）
- **發展邏輯**（≤50 字，他從哪來、怎麼走到現在、往哪去）：
- **職涯轉折點**（最關鍵的一個）：
- **他的下一步**（在為什麼目標鋪路）：
- **核心痛點**（他還沒解決、會有感的那一件事）：
- **建議切入角色**（我們以什麼身份出現在他的路徑上）：
- **個人化鉤子**（連線邀請最能讓他有感的具體切入點）：

## 【連線邀請】 ≤200 字 → custom1
[必含具體鉤子；禁字：介紹/產品/合作機會/方案]

## 【第一封 Day 1】 ≤300 字 → custom2
[行業現象 → 請教一個他願意答的問題 →「純粹好奇，沒有要推銷」]

## 【第二封 有回覆版 Day 5-7】 ≤300 字
[複述他的觀點 → 給低門檻資訊 → 讓他說「好」]

## 【第二封 無回覆追蹤版 Day 5-7】 ≤250 字 → custom3（Expandi Day7）
[輕、不施壓、換角度再試一次、給優雅出口]

## 【第三封 引入方案 Day 8-12】 ≤350 字
[從他發展邏輯的下一步切入 → AI Token King 的設計恰好對接 → 邀 20 分鐘實機操作]
```

---

**發送前檢查**：三封信都過了三自問？個人化鉤子是具體的、不是通用的？長度都在限制內？
若任一筆資訊不足以做出真個人化 → 標記「需補：[具體資訊]」，不要用通用話術硬寫。
