## 狀態標籤完整定義

| 標籤 | 說明 | 進入觸發 |
|-----|------|---------|
| LEAD-NEW | A-LI-Scout 產出，等待 Kid 審核 | Scout 自動建立 |
| LEAD-APPROVED | 通過 Gate 0，等待發送 | Kid 確認分類 |
| LEAD-REJECTED | 不符合 ICP 或審核不通過 | Kid 或 Scout 判定 |
| LEAD-SENT | 連結請求已發出 | Kid 核准並執行 |
| LEAD-NO-RESPONSE | 7 天後無任何回應 | Tracker 自動標記 |
| LEAD-CONNECTED | 對方已接受連結 | Tracker 偵測 |
| LEAD-ENGAGED | 有初步訊息往來 | Kid 判定為「有意義對話」|
| LEAD-HOT | 意願評分 4-5 分 | Kid Gate 1 判定 |
| LEAD-WARM | 意願評分 3 分 | Kid Gate 1 判定 |
| LEAD-COLD | 意願評分 1-2 分 | Kid Gate 1 判定 |
| DEMO-SCHEDULED | Demo 時間已確認 | Kid 確認後更新 |
| DEMO-DONE | Demo 已完成 | Kid 更新 |
| PROPOSAL-SENT | 報價單已寄出 | Kid 或 Alice 更新 |
| CLOSED-WON | 成交，移交 Yulenda | Kappa 確認收款 |
| CLOSED-LOST | 結案，附原因 | Kid Gate 4 判定 |
| NURTURE | 時機未到，設重啟條件 | Kid Gate 4 判定 |

## 階段一：陌生開發（Cold Outreach）

### 1.1 收到 A-LI-Scout Lead 後的三層過濾（目標：10 分鐘/人）

**第一層：基本資料核對（2 分鐘）**

```
✓ 公司規模符合 ICP（10~200 人）？
✓ 職稱為決策者本人（老闆 / IT 主管 / 總監）？
✓ 行業在目標清單內？
✓ LinkedIn 最後活躍在 30 天內？
✓ 是否已在 Asana CRM 中存在（避免重複觸達）？

→ 有任何一項不符合：標記 LEAD-REJECTED，附原因。
→ 全部通過：進入第二層。
```

**第二層：訊號掃描（5 分鐘）**

| 訊號層級 | 特徵 | 行動時機 | 分類 |
|---------|------|---------|------|
| Tier 1（主動購買訊號）| 近 30 天分享 AI 相關內容、JD 有 AI 關鍵字、詢問過類似方案 | 今天 | HOT |
| Tier 2（組織變動訊號）| 新任 IT/數位主管、近期融資擴編 | 本週 | HOT 或 WARM |
| Tier 3（行為/技術訊號）| 符合行業但無明確 AI 訊號 | 培育追蹤 | COLD |
| 無訊號 | 符合 ICP 基本條件但無訊號 | 低頻培育 | COLD |

**第三層：個人化確認（3 分鐘）**

```
✓ A-LI-Writer 草稿中引用的個人動態真實存在且時效在 14 天內？
✓ 公司名稱、職稱、姓名正確（無錯字）？
✓ 訊息語氣不過於銷售感（第一句不推產品）？
✓ 字數在 300 字以內？
✓ Call-to-Action 明確且低門檻？

→ 通過：進入 Gate 0 最終確認
→ 不通過：退回 A-LI-Writer 修改，附上修改方向
```

---

### 1.2 HOT / WARM / COLD 分類決策樹

```
Q1：是否有 Tier 1 訊號？
    ├── YES → [HOT] ★ 今天內聯繫
    └── NO  → Q2

Q2：是否有 Tier 2 訊號？
    ├── YES → Q3
    └── NO  → Q5

Q3：決策者是否為老闆本人或 C-level？
    ├── YES → [HOT] ★ 48 小時內聯繫
    └── NO  → [WARM] 本週聯繫

Q4（WARM 升降級判斷）：
    對方接受連結後 7 天內有主動發起對話？
    ├── YES → 升級為 [HOT]
    └── NO  → 維持 [WARM]

Q5：是否有 Tier 3 訊號？
    ├── YES → [COLD] 進入培育序列
    └── NO  → Q6

Q6：是否符合 ICP 基本條件但無任何訊號？
    ├── YES → [COLD] 低頻培育（每月 1 次）
    └── NO  → [LEAD-REJECTED] 結案

升級條件：COLD → WARM（對方開始互動 AI 內容）
          WARM → HOT（對方主動詢問方案 / 競品合約到期）
降級條件：HOT → WARM（30 天無任何回應）
          WARM → COLD（14 天無回應且無新訊號）
```

### Gate 0：送出連結請求前的最後核查（5 個問題，全 YES 才放行）

```
問題 1：這個人符合 ICP 定義嗎？
        （公司規模 10~200 人 + 行業相符 + 決策者本人）
        □ YES → 繼續 ｜ □ NO → 退回，標記 LEAD-REJECTED

問題 2：訊息中是否有至少一個真實的個人化細節？
        （引用對方近期貼文 / 公司動態 / 共同聯繫人）
        □ YES → 繼續 ｜ □ NO → 退回 A-LI-Writer 重寫

問題 3：如果我是對方，收到這訊息會覺得被打擾嗎？
        □ 不會被打擾 → 繼續 ｜ □ 感覺像廣告 → 退回修改語氣

問題 4：今天數量在控制範圍內嗎？（每日上限 15 筆）
        □ 在範圍內 → 繼續 ｜ □ 超過上限 → 排程到隔天

問題 5：Asana 是否已建立追蹤卡片，初始狀態設為 LEAD-SENT？
        □ YES → 放行送出 ｜ □ NO → 先建卡片再送出
```

## 階段二：初步接觸（First Contact）

### 2.1 連結請求送出後，三種狀態的處理

#### 狀態 A：對方接受連結

**A1 — 接受連結 + 有主動訊息**

```
狀態標籤：[LEAD-ENGAGED]
Kid 必須在 2 小時內回覆（訊號最熱時段）

判斷意圖（30 秒）：
  → 明確詢問方案/定價：直接進入購買意願評分（Gate 1）
  → 禮貌性回覆（謝謝 / 好的）：進入 A2 處理

回覆框架：
  「[姓名] 您好，很高興認識！
   看到您們在 [具體事業方向]，
   我最近在服務幾家類似規模的 [同行業] 客戶，
   在 AI Token 管理這塊有些心得想跟您分享。
   請問您這週有沒有 15 分鐘，
   讓我簡單說明我們怎麼幫他們解決 [對應痛點]？」

Asana 更新：對方訊息原文 + 判斷意圖 + 下一步動作 + 預計跟進日期
升級觸發：對方第一封訊息就詢問定價 → 立即升級為 [LEAD-HOT]
```

**A2 — 接受連結 + 無主動訊息**

```
狀態標籤：[LEAD-CONNECTED]
等待 24 小時，觀察對方是否自主發起對話

24 小時後無主動訊息 → Kid 發送 Touch 3：
  「[姓名] 您好，感謝接受連結！
   剛看了您分享的 [最近一篇貼文或公司動態]，
   [一句真誠的個人看法，15 字以內]。
   
   我在協助幾家 [行業] 的企業優化 AI 使用成本，
   如果您近期有類似的課題，歡迎交流想法。
   祝一切順利！」

等待 72 小時：
  → 有回覆 → 購買意願評分（Gate 1）
  → 無回覆 → 進入狀態 C 的無回應流程
```

#### 狀態 B：對方已讀但未接受（48 小時內）

```
狀態標籤：維持 [LEAD-SENT]，備註「已讀未接受」
等待 48 小時靜默，不追加任何動作

48 小時後判斷：
  → HOT Lead：考慮點讚對方近期貼文（增加曝光，不主動追訊）
  → WARM/COLD Lead：進入 Day 7 無回應流程（狀態 C）

注意：不在 LinkedIn 上重複發送連結請求（避免被標記騷擾）
```

#### 狀態 C：對方完全沒有動作（7 天後）

```
狀態標籤：[LEAD-NO-RESPONSE]

Kid 的決策邏輯：
  Step 1 — 確認對方 LinkedIn 在這 7 天是否有活躍跡象
    → 有活躍（有發文/按讚）：對方可能選擇不接受，降低優先級
    → 無活躍（帳號安靜）：可能真的沒看到

  Step 2 — 是否嘗試備用渠道？
    → HOT Lead 且有公司 Email：Day 10 透過 Email 輕觸碰一次（新話題開場）
    → WARM/COLD Lead：進入 Day 14 培育節奏

  Step 3 — 更新 Asana：現況、活躍度觀察、下一步計畫與日期
```

## 階段三：分類追蹤（Classification & Tracking）

### 3.1 初次回覆後 5 分鐘內「購買意願評分表」（Gate 1）

| 分數 | 行為特徵 | 分類 | 後續節奏 |
|-----|---------|------|---------|
| 5 分 | 主動詢問定價/方案、要求 Demo、提到競品不滿意 | LEAD-HOT | 當天排進 Demo 流程 |
| 4 分 | 詢問功能/案例、回覆超過 3 句且有具體問題 | LEAD-HOT | 48 小時內安排 Discovery Call |
| 3 分 | 「好的，我先了解看看」、索取資料但不約時間、轉介給 IT 部門 | LEAD-WARM | 發送案例後 3 天追蹤 |
| 2 分 | 「現在比較忙，之後再說」、非常簡短回覆 | LEAD-COLD | 降頻追蹤（每 30 天一次）|
| 1 分 | 明確拒絕、已有長期合約、12 個月內不考慮 | NURTURE 或 CLOSED-LOST | 見 3.3 結案標準 |

### 3.2 無回應者的追蹤節奏（Day 3 / 7 / 14 / 30 / 45）

**Day 3** — 分享有價值的內容（非銷售）
```
「[姓名] 您好，剛看到一篇關於 [相關行業趨勢] 的報告，
 想到您們在 [業務方向]，或許有點參考價值，分享給您。
 [附上文章連結]」
目的：保持存在感，不要求回覆
```

**Day 7** — 輕量詢問，降低門檻
```
「[姓名] 您好，之前分享的內容不知道有沒有機會看到。
 我想問個很簡單的問題：
 目前您們在 AI API 費用管理上，最讓您頭痛的是什麼？
 （不用長篇，一個詞也 OK）」
等待：72 小時
```

**Day 14** — 分享同行業成功案例（具體數字）
```
「[姓名] 您好，我知道您最近很忙，打擾一下。
 上週我們剛完成了一家 [同行業] 客戶的導入，
 在 AI Token 費用上降低了 38%，
 3 個月內的 ROI 計算方式我整理成一頁表格，
 如果您有興趣，我寄給您參考。」
等待：5 個工作天
```

**Day 30** — 再活化（換角度切入）
```
選擇 A（HOT 轉來的沉默 Lead）：
  「[姓名] 您好，距離上次聯繫已有一段時間了。
   我沒有要推銷任何東西，
   只是想問：現在是不是還不是合適的時間點？
   如果是的話，請告訴我什麼時候比較方便，我到時候再聯繫。」

選擇 B（WARM/COLD Lead）：
  「[姓名] 您好，我們剛上線了一個新功能 — [具體功能]，
   對 [對方行業] 在 [具體場景] 上幫助很大。
   如果有機會，想花 10 分鐘聊聊是否符合您們的需求。」

選擇依據：曾有任何互動記錄 → 選擇 A；從未互動 → 選擇 B
```

**Day 45** — 最後一觸 + 結案決策
```
「[姓名] 您好，這是我最後一次主動聯繫，不想繼續打擾您的時間。
 如果未來在 AI Token 管理或 AI 工具整合上有任何需要，
 歡迎隨時找我，我的聯繫方式都在這裡。
 祝一切順利！— Kid / AI Token King」

決策：見 3.3
```

### 3.3 NURTURE vs CLOSED-LOST 判斷標準

**標記為 [NURTURE]（值得培育，未來有機會）：**
```
✓ 對方明確說「現在不是時機，但未來可能有興趣」
✓ 公司目前方案合約在 6~12 個月內到期
✓ 對方個人持續關注 AI 工具相關內容
✓ 公司規模正在成長（有招聘 / 有新融資）
✓ 45 天序列無回應，但 LinkedIn 仍活躍

培育頻率：每 60 天一次輕觸碰（分享相關內容或祝賀里程碑）
重新評估時機：對方主動互動、Tier 1 訊號出現、公司有重大變動
```

**標記為 [CLOSED-LOST]（真正無需求，結案）：**
```
✓ 明確拒絕，且表示 12 個月內無相關預算
✓ 公司規模不符合 ICP（後來發現是工作室或大型企業）
✓ 行業確認無法受益於 AI Token 管理
✓ 已有深度合作的競爭對手且滿意度高
✓ 45 天序列走完，LinkedIn 帳號已停止活躍（可能離職）

結案後重新觸達窗口：
  CLOSED-LOST（拒絕型）：最少 6 個月
  CLOSED-LOST（無回應型）：最少 3 個月
  NURTURE：60 天

重新觸達觸發條件（滿足任一即可）：
  → 對方換公司或升職
  → 對方公司新一輪融資或擴張
  → AI Token King 推出重大新功能
  → 市場發生大規模 AI 成本事件（GPT/Claude 調漲等）
  → 競品合約到期（若有記錄）
  → 對方在 LinkedIn 發佈 AI 成本相關貼文
  → 共同聯繫人提到對方近期有相關需求
```

## 階段四：合作回應（Response Handling）

### 4.1 有回應的路徑

#### 正面回應（有興趣）→ 2 步驟內排到 Demo（Gate 2）

```
Step 1 — 收到正面回應後 2 小時內回覆，先確認兩件事：

  a. 對方是否為最終決策者？
     → 是：直接邀約 Demo
     → 不是：「請問在這個決策上通常還有誰需要一起了解？
             如果可以的話，是否方便邀請他/她一起加入通話？」

  b. 了解一個核心痛點（客製化 Demo 內容用）：
     「在我準備展示之前，想先請教：
      您目前在 AI API 使用上，最想解決的是費用控制、
      使用量追蹤、還是多人協作管理這一塊？」

Step 2 — 確認 Demo 時間（同一封訊息完成）：
  「我這週 [週二下午 / 週四上午] 有空，
   或者您告訴我您方便的時段，
   我安排一個 20 分鐘的線上 Demo，
   讓您直接看到平台操作和我們客戶的實際數字。」

確認後即時動作：
  → 發送 Google Meet / Zoom 邀請
  → Asana 更新狀態為 DEMO-SCHEDULED
  → 準備客製化 Demo 重點（根據對方痛點）
  → Demo 前一天發提醒訊息
```

**Demo 前一天提醒訊息：**
```
「[姓名] 您好，提醒一下明天 [時間] 的線上會議。
 連結：[Meet連結]
 我會準備 [對方提到的痛點] 相關的案例重點。
 如有任何問題或需要更改時間，隨時告訴我。」
```

---

#### 中性回應（有問題/要更多資訊）→ 轉化為進一步對話

```
核心策略：不直接寄 PDF（資料會被忽略），先問問題再給精準資訊

Step 1 — 接住問題，反問一個問題：
  範例：對方說「可以介紹一下你們的功能嗎？」
  
  回應：「當然可以！AI Token King 的功能涵蓋蠻廣的，
         為了給您最相關的資訊，
         請問您目前主要是在哪個情境下用 AI API？
         （比如：內部工具、客服自動化、還是資料分析？）」

Step 2 — 根據對方回答，給精準說明 + 一個邀約：
  「根據您說的 [情境]，我們有幾個客戶用在類似場景，效果蠻具體的。
   這樣子 —— 我傳一個 2 分鐘的產品短片給您，
   看完如果覺得有意思，我們再約 15 分鐘聊細節，如何？」

轉化判斷：
  → 2 個來回後仍只要資料不願意約時間 → 評為 3 分 WARM
  → 繼續執行 Day 3 追蹤節奏，不強行推進
```

---

#### 負面回應（現在不需要 / 預算問題）→ 優雅結案保留機會

| 類型 | Kid 的回應策略 | 後續動作 |
|-----|-------------|---------|
| 「現在沒有預算」| 「完全理解。我想確認一下，是今年確定沒有預算，還是 Q3/Q4 之後可能重新評估？（如果後者，我到時候帶最新方案給您參考）」| 取得具體時間點 → 標記 NURTURE，設提醒 |
| 「已有其他工具了」| 「了解，能問一下是哪個方向的工具嗎？（純粹市場調研，不是要說服您換）有時候不同工具可以互補。」| 記錄競品資訊 → 若合約 6-12 個月後到期 → NURTURE |
| 「現在太忙了」| 「完全理解。我們把導入設計成 2 個小時可以完成的流程。這樣子 —— 等您忙完這陣子，您主動告訴我一聲就好，我這邊先把您的資料存著。」| 標記 NURTURE，30 天後再輕觸碰一次 |

**所有負面回應的結語原則：**
- 永遠不爭論，不說「但是我們比較好」
- 給對方一個乾淨的出口
- 讓對方記得你是「有禮貌、不黏人、但隨時都在」的那個人

### 4.2 無回應的路徑（Day 14 / 30 / 45 決策）

**Day 14 無回應 — 策略選擇：**
```
先查看 A-LI-Tracker 這 14 天的行為數據：

若對方有 LinkedIn 活躍（發文/按讚）：
  → 換角度切入（不提之前訊息，用全新話題）
  → 「[姓名] 您好，看到您最近分享了 [最新貼文話題]，
      想到一個問題請教您：[與他工作相關的開放式問題]」

若對方完全無活躍：
  → 嘗試 Email 渠道（若有）
  → 訊息聚焦「一個具體數字」
  → 附低門檻 CTA：「如果您有興趣，回個 YES，我發詳細資料給您」
```

**Day 45 最終決策框架（3 分鐘評估）：**
```
問題 1：這 45 天內，對方有任何 LinkedIn 活躍跡象嗎？
  → 完全無活躍 + 無法確認是否還在職 → [CLOSED-LOST]，備註「帳號可能廢棄」

問題 2：公司基本面在這 45 天有任何正面變化嗎？
  （新融資 / 擴編 / 新主管 / 相關新聞）
  → 有正面變化 → Last Touch 後標記 [NURTURE]（60 天後重新評估）
  → 無變化 → 標記 [CLOSED-LOST]

問題 3：我對這個 Lead 還有直覺上的信心嗎？
  → 有，且能說出具體原因 → [NURTURE]
  → 沒有，純粹是捨不得放棄 → [CLOSED-LOST]（克服沉沒成本）
```

## 關鍵決策點（Gate）設計

### Gate 0 — 送出連結請求前（LEAD-APPROVED → LEAD-SENT）

| 項目 | 說明 |
|-----|------|
| 執行者 | Kid（人工確認）|
| 標準 | 1.3 五個核查問題全部 YES |
| 耗時 | 2~3 分鐘/筆 |
| 失敗動作 | 退回修改或標記 LEAD-REJECTED |

---

### Gate 1 — 初步回應品質判斷（LEAD-ENGAGED → 意願分級）

| 項目 | 說明 |
|-----|------|
| 執行者 | Kid（5 分鐘內完成）|
| 標準 | 購買意願評分完成（1-5 分）|
| 判斷輸出 | HOT / WARM / COLD 標籤 + 後續節奏決定 |
| 失敗動作 | 無法判斷 → 預設 3 分 WARM |

---

### Gate 2 — Demo 準備確認（LEAD-HOT → DEMO-SCHEDULED）

| 項目 | 說明 |
|-----|------|
| 執行者 | Kid（準備）+ Alice（技術問題後備）|
| 標準 | ① 確認對方為決策者或決策鏈已清楚 ② 了解至少一個核心痛點 ③ Demo 材料已準備對應場景 ④ 會議邀請已發送且對方已確認 |
| 失敗動作 | 對方為非決策者 → 先請對方邀請決策者再排 Demo |

---

### Gate 3 — 提案送出前確認（DEMO-DONE → PROPOSAL-SENT）

| 項目 | 說明 |
|-----|------|
| 執行者 | Kid（確認資訊）+ Alice（報價）+ 依金額決定簽核層級 |
| 標準 | ① Demo 後有明確正面意願（口頭確認）② 預算範圍已初步確認 ③ 決策時間線已了解 ④ 報價單由 Alice 確認金額正確 ⑤ 金額 > NT$50,000 → Lauren 簽核；> NT$200,000 → Jet 簽核 |
| 失敗動作 | 對方 Demo 後態度不明確 → 進入中性回應處理流程 |

---

### Gate 4 — 結案確認（任何階段 → CLOSED-LOST 或 NURTURE）

| 項目 | 說明 |
|-----|------|
| 執行者 | Kid（判斷）+ Lauren（知會，每週匯報）|
| 標準 | CLOSED-LOST：符合 3.3 結案條件 ｜ NURTURE：有明確培育價值且設定好重啟條件 |
| 要求 | 每筆 CLOSED-LOST 必須在 Asana 附上至少一句結案原因 |
