## 背景說明：為什麼要分本機和雲端

雲端容器（Claude Code）的對外 IP 被 LinkedIn 偵測為機器人流量，直接操作 LinkedIn 的動作一律會被封鎖。所以我們把工作切成兩塊：

**本機（Kid 的電腦）負責：**
- Sales Navigator 搜尋、查看 Profile
- 連結請求、發送訊息
- 收件夾管理和回覆
- 執行 apify_linkedin_scraper.py 抓取名單

**雲端（Claude Code）負責：**
- 讀取 CSV，做 ICP 複核和評分
- 生成個人化訊息草稿
- 產出 Asana 匯入格式
- 分析回覆、給下一步建議

記住這個原則：**雲端想，本機做。** 所有實際按下送出的動作，都在你自己的電腦上完成。

## Part 1：每日作業清單（Kid 本機執行）

> 每天總時間投入：約 90 分鐘（分三個時段）
> 原則：按優先順序做，做不完的先做高分 HOT Lead。

---

### 早班（8:30 ~ 9:00）— 30 分鐘，15 個具體動作

這個時段是每天最重要的 30 分鐘。你在這裡決定今天的優先順序，晚一步訊號就涼了。

**開啟 Sales Navigator，依序執行以下動作：**

**動作 1 — 看昨晚進來的 Lead & Account Alerts（3 分鐘）**

點左側選單「Alerts」，依照 Tier 分類處理：

| Tier | 代表什麼 | 今天要怎麼做 |
|------|---------|------------|
| Tier 1 | 對方最近 30 天有發 AI 工具相關貼文、公司在徵 AI 工程師 | 今天優先觸達，不能等明天 |
| Tier 2 | 新主管上任、公司獲得融資消息 | 今天內要行動，最晚明早 |
| Tier 3 | 符合 ICP 但無明確訊號 | 加入 Lead List 觀察即可 |

在 Alerts 頁面，Tier 1 先截圖或複製對方姓名備用（等等要帶入訊息草稿）。

**動作 2 — 查看「Who's Viewed Your Profile」（2 分鐘）**

Sales Navigator 首頁右上角「Who Viewed Your Profile」點進去，確認過去 24 小時誰看過你的 Profile。

- 符合 ICP 的人 → 馬上點進他的 Profile 查看，今天就發連結請求
- 不符合 ICP 的人 → 不用理會

**動作 3 — 確認昨日連結請求的狀態（3 分鐘）**

進入「Connections」，篩選「Pending」，查看昨天發出的連結請求。

- 對方已接受：把這個人的名字記下來，等一下發 Touch 3（首次跟進訊息）
- 對方接受且主動傳訊息：立刻升為最高優先，2 小時內回覆
- 7 天以上未接受的：考慮改用 InMail，或先結案到 Nurture

**動作 4 — 打開 Asana，確認今天的觸達節點任務（3 分鐘）**

篩選「Due Date = 今天」的任務，確認哪些 Lead 今天要做哪個 Touch：

- Day 3 → Touch 3（首次跟進）
- Day 7 → Touch 4（價值觸達）
- Day 14 → Touch 5（Demo 邀約）
- Day 30 → Touch 6（再活化）
- Day 45 → Touch 7（最後一觸）

把今天要執行的 Touch 任務清單抄到便利貼或筆記本，等一下一筆一筆執行。

**動作 5 — 確認今天有沒有排定的 Demo（1 分鐘）**

打開 Google Calendar，確認今天有沒有 Demo 或 Discovery Call。有的話，提前 15 分鐘準備好對方資料和客製化重點。

**動作 6 — 執行 apify_linkedin_scraper.py 抓取今日新 Lead（10 分鐘）**

打開終端機，執行以下步驟。第一次執行要多 5 分鐘裝套件，之後就快很多。

步驟 A — 確認套件已安裝（第一次執行才需要）：
```
pip install apify-client python-dotenv
```

步驟 B — 確認 .env 檔存在，兩個環境變數都有填：
```
APIFY_TOKEN=你的_apify_token
LINKEDIN_LI_AT=你的_linkedin_cookie
```

.env 檔放在 `scripts/` 目錄下，不要 commit 到 git。

步驟 C — 執行腳本：
```
cd projects/aitokenking/sales/scripts
python apify_linkedin_scraper.py
```

腳本跑完會輸出一個摘要，告訴你今天抓到幾筆 HOT / WARM / COLD Lead，同時在 `scripts/output/` 底下生成 `leads_YYYYMMDD_HHMMSS.csv`。

步驟 D — 看完摘要，把 CSV 路徑複製備用，等一下要傳給 Claude Code 做複核。

注意：腳本會跑三個搜尋組合（組合 A / B / C），每組之間等 30 秒，總時間約 5~8 分鐘。趁這段時間可以繼續做下面的動作。

**動作 7 — 處理昨日 Alert：檢查有沒有「Job Change」類型（2 分鐘）**

在 Alerts 中找「Job Change」類型的 Alert，這是最高價值的訊號之一。

對方換新工作或升職 → 代表他進入新環境，有新的預算需求 → 這是 Tier 2 訊號，今天內要觸達。

如果這個人已經在你的 CRM 裡，直接到 Asana 找到他的 Task，把狀態從 NURTURE 或 COLD 改回 WARM，今天重新發訊息。

**動作 8 — 回覆昨晚 8 點以後進來的 LinkedIn 訊息（3 分鐘）**

打開 LinkedIn Messaging，篩選時間，先看有沒有昨晚下班後進來的訊息。上班後的第一件事把這些回完，回覆率和速度是對方信任感的關鍵。

**動作 9 — 確認昨天 Touch 3 之後有沒有對方開啟了 Smart Link（1 分鐘）**

如果你上週有傳 Smart Links 給對方（產品介紹、案例文件），Sales Navigator 首頁會顯示誰點開了。點開但沒回覆的人 → 今天是跟進的好時機。

**動作 10 — 查看「Saved Searches」有沒有新的 Lead 推薦（2 分鐘）**

Sales Navigator 左側「Searches」，查看三個 Saved Search（組合 A / B / C）旁邊有沒有數字，有數字代表有新的符合條件的 Lead。點進去看一眼，決定要不要加入今天的觸達清單。

這個動作每天看一次就夠，不用每次都把新 Lead 全部立刻行動，看自己今天的時間決定。

**動作 11 — 把 CSV 傳給 Claude Code 做 ICP 複核（以下為雲端步驟，但從本機發起）**

在你的終端機或 Claude Code 對話視窗，輸入：

```
@claude 請幫我複核以下 CSV 的 ICP 評分，生成個人化訊息草稿，輸出 Asana 匯入格式。
CSV 路徑：projects/aitokenking/sales/scripts/output/leads_YYYYMMDD_HHMMSS.csv
今天日期：2026-05-30
重點行業：科技公司 IT 決策者（組合 A）優先
```

雲端處理完之後，會輸出：
1. 每筆 Lead 的複核評分結果（修正前置腳本可能的誤判）
2. 每筆 HOT Lead 的個人化訊息草稿（連結請求用）
3. Asana 匯入格式的 CSV（可直接在 Asana 批次建立 Task）

這個步驟啟動後，你可以繼續做下面的動作，Claude Code 在背景跑。

**動作 12 — 執行今天的連結請求（依 Asana 清單）（5 分鐘）**

打開昨晚 Claude Code 幫你準備的訊息草稿（存在 Google Docs 或貼在 Asana 任務描述裡），逐一在 Sales Navigator 找到對應的人，發連結請求。

每筆動作順序：
1. Sales Navigator 搜尋對方姓名
2. 確認是本人（看公司和職稱）
3. 點「Connect」
4. 選「Add a note」
5. 貼入草稿，確認個人化細節（名字、公司名是否正確）
6. 送出

每天上限 15 則連結請求，不要超過，超過會觸發 LinkedIn 帳號限制。

**動作 13 — 對今天 Alert 裡的活躍 Lead 留評論（3 分鐘）**

找到 Tier 1 / Tier 2 Alert 中，有在 30 天內發文的 Lead，對他們的最新貼文留一則真誠的評論。每天最多 3 則，不要更多。

評論要真誠，不要廣告感。範例：
「這個點觀察得很準，AI 工具的採購決策確實越來越複雜，我們也在幫一些團隊梳理這塊。」

這個動作的目的是讓對方在你發連結請求之前就對你有印象，接受率會高很多。

**動作 14 — 把今天的 InMail 使用情況記到便利貼（1 分鐘）**

每月 50 封 InMail 上限。今天用了幾封，記下來。銷售月初可以比較大方用，月底留 10 封備用。

**動作 15 — 早班結束前更新 Asana（1 分鐘）**

把早班做的所有動作（連結請求已發、評論已留）更新進 Asana 對應的 Task，設定各 Task 的下一個觸達日期。

---

### 午班（12:00 ~ 12:30）— 30 分鐘

這個時段主要做兩件事：跟進昨日已連結但沒觸達的 Lead，以及確認 Claude Code 準備好的今日訊息草稿。

**步驟 1 — 確認昨日已接受連結但還沒發 Touch 3 的人（10 分鐘）**

在 Asana 篩選「狀態 = LEAD-CONNECTED，Touch 編號 = D1-待發，連結接受日期 = 昨天」的 Task，這些人是今天 Touch 3 的目標。

到 LinkedIn Messaging，逐一發送首次跟進訊息（模板 2 A 版或 B 版），記得替換：
- [對方名字]
- [公司名]
- [最近一篇貼文或公司動態]（這個細節最重要，跳過它就只是群發）

發完後，在 Asana 把 Touch 編號更新為 D3-已發，設定 3 天後提醒。

**步驟 2 — 確認 A-LI-Writer 草稿並手動發送（15 分鐘）**

Claude Code 今早應該已經跑完複核，打開你們的對話或 Google Docs，確認草稿：

審核草稿的三個重點：
1. 個人化細節是否真實（貼文引用是否存在且時效在 14 天內）
2. 第一句話有沒有銷售感（有的話，改掉）
3. 字數是否在 300 字以內

通過的草稿直接複製去 LinkedIn 發送；有問題的草稿，直接在 Claude 對話裡說「這一筆的貼文引用不對，請改用對方的公司最近的招聘 AI 工程師這個訊號重新寫」。

**步驟 3 — 快速確認收件夾有無新進訊息（5 分鐘）**

掃一遍 LinkedIn Messaging，有新回覆的先標記分類（詳見 Part 3），確認有沒有需要今天內回覆的緊急訊息。

HOT 回覆（對方詢問方案或定價）→ 必須今天內回，最晚 2 小時。

---

### 晚班（17:30 ~ 18:00）— 30 分鐘

收尾和準備明天，不做重大決策，做的是確認和記錄。

**步驟 1 — 處理今日所有未回覆的 LinkedIn 收件（10 分鐘）**

打開 LinkedIn Messaging，把今天所有新進訊息都回完。

依照 Part 3 的分類腳本逐一處理，不要讓任何訊息在當天沒有回覆。

**步驟 2 — Asana 狀態全面更新（10 分鐘）**

今天做過的每一筆動作，確認都更新在 Asana 對應 Task 上：

- 今天發了連結請求的 → 狀態改為 LEAD-SENT，設定 7 天後提醒
- 今天有人接受連結的 → 狀態改為 LEAD-CONNECTED，設定明天提醒（發 Touch 3）
- 今天有回覆且有意義的 → 狀態改為 LEAD-ENGAGED，填購買意願分數（1-5 分）
- 今天確認 Demo 的 → 狀態改為 DEMO-SCHEDULED，Google Calendar 建立會議
- 今天結案的 → 狀態改為 CLOSED-LOST 或 NURTURE，必須附一句結案原因

每天下班前，Asana 裡不能有任何「下次觸達日期」空白的 Task。

**步驟 3 — 準備明天觸達清單（10 分鐘）**

在 Asana 篩選「Due Date = 明天」的 Task，看一下明天要做哪些 Touch。

對於明天要做 Touch 4（價值觸達）和 Touch 5（Demo 邀約）的 Lead，今天晚上就先把草稿請 Claude Code 準備好：

```
@claude 請幫我準備明天的觸達草稿：
- [姓名] | Touch 4，訊號：對方公司有在招聘 Prompt Engineer
- [姓名] | Touch 5，之前交流提到帳單管理是痛點，職稱 IT Director
```

Claude Code 會在你下班後準備好，明天早班直接用。

## Part 2：Lead 抓取 → 雲端處理 → 本機執行的完整流程

這個循環是整個系統的心臟，每天跑一次。搞懂這個，新人第一週就能上手。

```
[本機] Kid 執行腳本
       ↓
[本機] 產出 leads_YYYYMMDD.csv
       ↓
[指令] Kid 把 CSV 傳給 @claude
       ↓
[雲端] Claude Code 做 ICP 複核 + 生成訊息草稿
       ↓
[雲端] 輸出 Asana 匯入格式 + 訊息草稿
       ↓
[本機] Kid 在 LinkedIn 手動執行連結請求 + 發訊息
       ↓
[本機] 收到回覆後，截圖或複製文字傳給 @claude
       ↓
[雲端] Claude Code 分析回覆，給下一步建議 + 訊息草稿
       ↓
       回到本機執行
```

---

### 步驟 1：本機執行腳本，產出 CSV

**執行者：** Kid 本機
**時間：** 早班 8:30~8:40，約 8~10 分鐘（含腳本跑完等待）

```bash
cd projects/aitokenking/sales/scripts
python apify_linkedin_scraper.py
```

**輸入：** .env 裡的 APIFY_TOKEN 和 LINKEDIN_LI_AT
**輸出：** `scripts/output/leads_YYYYMMDD_HHMMSS.csv`（每筆 Lead 含名字、職稱、公司、LinkedIn URL、ICP 分數）

腳本跑三個搜尋組合，中間等 30 秒。跑完之後終端機會印出今天的摘要：
- 總 Lead 數（去重後）
- HOT / WARM / COLD 各幾筆
- Top 5 HOT Lead 清單

先把 CSV 完整路徑複製起來。

---

### 步驟 2：把 CSV 傳給 Claude Code 做 ICP 複核和訊息草稿生成

**執行者：** Kid 透過 Claude Code 對話介面
**時間：** 早班 8:40~8:50 啟動，Claude 在背景跑，約 3~5 分鐘完成

在 Claude Code 對話中輸入：

```
@claude 我今天抓到新的 Lead 名單，請做以下處理：

1. 讀取 CSV：projects/aitokenking/sales/scripts/output/leads_20260530_083542.csv
2. 依照 linkedin-cold-outreach-sop.md 的 ICP 定義做複核，
   修正腳本自動評分可能誤判的 Lead（尤其是職稱切合度）
3. 對所有 HOT Lead（≥70分）生成個人化連結請求訊息草稿（模板 1 A 版或 B 版）
4. 輸出 Asana 匯入格式（含 Task 名稱、ICP 分數、分類、Priority、初始狀態）

今天優先：組合 A 科技公司 IT 決策者，有 AI 相關貼文的優先。
```

**輸入：** `leads_YYYYMMDD_HHMMSS.csv`
**輸出：**
- 複核後的 ICP 分類清單（標記調整原因）
- 每位 HOT Lead 的個人化訊息草稿（直接可貼進 LinkedIn）
- Asana 匯入用 CSV（批次建立 Task）

---

### 步驟 3：Claude Code 輸出 Asana 匯入格式 + 訊息草稿

**執行者：** Claude Code 雲端
**時間：** 步驟 2 啟動後約 3~5 分鐘

Claude 會在對話中直接輸出：

**格式 A — Asana 匯入 CSV（可直接上傳到 Asana）：**

| Task Name | Section | Assignee | Due Date | ICP Score | Classification | Priority | LinkedIn URL |
|-----------|---------|----------|----------|-----------|----------------|----------|-------------|
| [HOT-85] 王大明 \| IT Manager \| XYZ科技 \| 85分 | 00 Lead Pool | Kid | 2026-05-30 | 85 | HOT | https://... |

**格式 B — 訊息草稿（每人一段）：**

```
[對象：王大明 | XYZ科技 IT Manager | ICP 85分]
草稿類型：連結請求（模板 1A）
個人化依據：對方昨天發文提到評估 Claude API 的成本結構

草稿內容：
王大明您好，看到您昨天分享了 Claude API 成本結構的觀察，
這個議題我們最近也在幫幾家科技公司做整合規劃。
想跟您交流一下行業的觀察，加個好友方便嗎？
```

確認草稿沒問題後，在 Asana 把 Asana 匯入 CSV 上傳建立 Task，訊息草稿複製到對應 Task 的描述欄位備用。

---

### 步驟 4：Kid 在 LinkedIn 手動執行連結請求和發訊息

**執行者：** Kid 本機
**時間：** 早班 8:50~9:00，午班 12:00~12:15，各約 10~15 分鐘

打開 Sales Navigator，逐一找到 Claude Code 草稿清單裡的人，發送連結請求。

操作流程：
1. Sales Navigator 搜尋對方完整姓名
2. 確認公司和職稱一致（防止找到同名的人）
3. 點「Connect」→「Add a note」
4. 貼入草稿，再過一遍：名字對嗎？公司名對嗎？引用的貼文是真的嗎？
5. 確認沒問題，點「Send」
6. 在 Asana 更新 Task 狀態為 LEAD-SENT，記錄發送時間

每天上限 15 則，不要超過。

---

### 步驟 5：收到回覆後，把回覆傳給 Claude Code 做分析

**執行者：** Kid 本機 → 傳給 Claude Code → 本機再執行
**時間：** 收到回覆後即時處理，最晚 2 小時內（HOT 回覆）

當對方回覆你的訊息之後：

做法 A（快速）：直接複製對方訊息文字，貼給 Claude Code：

```
@claude 我收到一封 Lead 回覆，請幫我分析並給建議：

Lead 資料：王大明 | XYZ科技 IT Manager | ICP 85分 | Touch 3
對方回覆：
「謝謝，最近確實在評估 AI 工具的整合方案，我們用了 OpenAI 和 Anthropic 兩家的 API，
管理起來有點複雜，你們的方案是怎麼幫忙解決的？」

請判斷：
1. 購買意願分數（1-5分）
2. 分類（HOT/WARM/COLD）
3. 這個回覆最適合用哪個應對腳本
4. 生成下一封訊息草稿
```

做法 B（圖片）：收到回覆後直接截圖，把截圖貼給 Claude Code，說明對方是誰。

**Claude Code 的輸出：**
- 購買意願分數 + 分類判斷 + 判斷依據
- 應對腳本建議（對應 `linkedin-crm-tracking-flow.md` 的回覆腳本）
- 下一封訊息草稿（草稿 Ready 後 Kid 複製去發）
- Asana 更新建議（狀態要改成什麼）

Kid 確認草稿沒問題，複製到 LinkedIn 回覆，更新 Asana。

## Part 3：訊息接收與回覆處理（收件夾管理）

LinkedIn Messaging 是你最重要的工作陣地。收件夾亂，業務就亂。

---

### 每日 LinkedIn Inbox 處理 SOP

**早班 8:50 掃一次，午班 12:20 掃一次，晚班 17:45 掃一次。** 每次 5 分鐘內完成分類，不要在這裡花太多時間想。

**快速掃描步驟：**

1. 打開 LinkedIn Messaging
2. 從最新訊息往下看
3. 每封訊息快速判斷類型（30 秒內決定）
4. 在訊息旁邊標記（LinkedIn 有星號功能，或自己在 Asana 備注）

**分類標記方式：**

| 符號 | 類型 | 今天要做什麼 |
|-----|-----|------------|
| ★★★ 紅星 | 正面回覆：對方問方案、價格、功能 | 2 小時內回，參考 Demo 邀約腳本 |
| ★★ 黃星 | 中性回覆：「好的」「謝謝」「我了解一下」 | 今天內回，先問一個問題 |
| ★ 灰星 | 婉拒：明確說不需要 | 用優雅結束腳本，更新 Asana |
| 無標記 | 已讀不回的 | 不用特別標記，看 Day 7/14 提醒再處理 |

---

### 正面回覆 — 2 步驟 Demo 預約腳本

對方說「有興趣」「可以介紹一下」「我們確實有這個問題」，這是你最重要的 10 分鐘。

**Step 1 — 收到正面回覆後，先確認一件事（先不急著約時間）：**

```
[對方名字] 您好，很高興您有興趣！

在我準備展示之前，想先問您一個問題：
您目前在 AI 工具管理上，最讓您頭痛的是哪一塊？

A. 帳單混亂，搞不清楚每個月花多少
B. 多人使用，沒有統一管控
C. 要向老闆說明 AI 投資有沒有效，但缺乏數據

（您也可以直接說，不一定要選其中一個）
```

對方回答之後，你就知道 Demo 要重點展示哪一塊，20 分鐘的時間絕對夠用。

**Step 2 — 對方回答後，馬上約時間：**

```
了解，[痛點 A] 確實是我們很多客戶遇到的狀況。

我這週 [週二下午 2-4 點] 或 [週四上午 10-12 點] 有空，
讓我用畫面跑一遍給您看，大概 20 分鐘，針對您說的 [痛點 A] 直接展示解法。

您哪個時段方便？或者您給我一個您方便的時間也可以。
```

確認時間之後，立刻：
- 發 Google Meet 邀請
- Asana 更新 DEMO-SCHEDULED
- 準備客製化 Demo 重點（這個動作可以請 Claude Code 幫你整理）

---

### 中性回覆 — 先問一個問題的 3 個範本

對方說「好的，我先了解看看」「有機會可以聊聊」「我轉給同事看」，不是拒絕，但也還沒行動。這種情況要用一個問題把對話繼續拉住。

**範本 A（了解現狀）：**
```
謝謝您 [對方名字]！

想先請教一個問題：您們現在團隊在用 AI 工具的話，
大概是讓大家自己各自管，還是公司有統一安排？

（不用說太精確，大概說個方向就好，我可以更清楚怎麼跟您說明）
```

**範本 B（確認痛點）：**
```
了解！

我想先確認一件事，這樣我給您的資訊才會更精準：
您們目前在 AI 帳單或用量管理這一塊，有沒有讓您比較頭痛的地方？

（就算現在還沒有也沒關係，我想知道現在的狀況怎麼樣）
```

**範本 C（給對方一個簡單選擇）：**
```
好的，謝謝！

我這邊有兩個方式可以繼續交流，您看哪個更方便：

A. 我傳一個 2 分鐘的產品短片給您，您有空再看
B. 我們先約 15 分鐘視訊，我直接講重點

您比較偏哪一個？
```

選項 A（傳短片）→ 這是降低門檻的做法，對方接受度高，但之後的跟進要快，3 天內追蹤對方有沒有看。
選項 B（約視訊）→ 成交率更高，但對方需要時間感。

---

### 負面回覆 — 3 種優雅結束對話的腳本

對方說「不需要」「現在沒有預算」「已經有其他工具」，這不是失敗，是在幫你節省時間。優雅結束，讓對方記得你是個有禮貌的人，以後有需要還是會想到你。

**腳本 A（對方說不需要）：**
```
謝謝您告訴我！完全理解。

如果之後在 AI 工具管理這塊有任何問題，或者有朋友遇到類似需求，
歡迎隨時找我，我很樂意幫忙。

祝您工作順利！— Kid
```

然後在 Asana 把這筆 Lead 狀態更新為 CLOSED-LOST，備注原因「對方說不需要，無進一步說明」。

**腳本 B（對方說已有其他工具）：**
```
了解，謝謝告訴我！

能問一下是哪個方向的工具嗎？（純粹市場調研，不是要說服您換）
因為我們有時候會跟其他工具互補，不一定是替代關係。

如果真的不適合也完全沒關係，就算認識一下業界朋友也很好。
```

這個腳本目的是獲取競品資訊。對方願意說就記下來；不願意說就直接進到腳本 A 的收尾。

**腳本 C（對方說現在太忙）：**
```
完全理解，現在事情多的時候真的很難有時間評估新工具。

這樣好了，我先把您的資料存著，等您這陣子忙完之後，
您覺得時機差不多了，主動傳訊給我就可以，我馬上回覆。

不會再主動打擾您，祝這陣子的工作順利！
```

這個腳本的好處是讓對方「主動找你」，比你再去追他的成功率更高。在 Asana 標記 NURTURE，設定 30 天後再輕觸碰一次。

---

### 已讀不回 — Day 7 / Day 14 再次跟進模板

對方接受了連結，你發了訊息，但他已讀不回。這很正常，別放棄，但也不要黏。

**Day 7 跟進（發送後第 7 天仍無回覆）：**

```
[對方名字] 您好，上次傳了訊息，可能時機不對或者您最近很忙。

只想問一個簡單的問題：
您們在 AI 工具的費用管理這塊，目前最讓您頭痛的是什麼？

（一個詞也 OK，就當交流一下業界狀況）
```

**Day 14 跟進（第 14 天仍無回覆）：**

```
[對方名字] 您好，打擾一下。

上週我們剛幫一家同行業的客戶完成整合，他們在 AI Token 費用上降低了 38%，
我把 3 個月的 ROI 試算整理成一頁表格，如果您有興趣，我寄給您參考。

不用任何回覆，如果感興趣就說一聲，我立刻寄過去。
```

Day 14 之後沒有回覆，就讓他進入 Day 30 和 Day 45 的節奏（按照 `linkedin-crm-tracking-flow.md` 的無回應流程走）。

## Part 4：LinkedIn Sales Navigator 進階操作技巧

### InMail 信用額度追蹤（每月 50 封上限）

Sales Navigator Professional 方案每月有 50 封 InMail，用完就要等下個月。

**追蹤方式：**

在 Asana 建一個 Task 叫「InMail 月度額度追蹤」，每次用掉一封就在 Task 描述裡記：

```
2026-05-30 | 王大明 | XYZ科技 IT Manager | 組合A | 已讀不回 7 天後用 InMail
2026-05-28 | 陳小明 | ABC行銷 Marketing Director | 組合B | 連結請求 7 天未接受
```

**使用節奏建議：**
- 月初 1-10 日：比較大方用，可以用 20 封
- 月中 11-20 日：節制一點，每週不超過 5 封
- 月底 21-31 日：保留 10 封備用，只給 HOT Lead

**哪種情境值得用 InMail：**
- 連結請求已發送 7 天以上，對方仍未接受，但 ICP 評分 ≥ 70 分
- 對方沒有「Open to connecting」設定（連結請求發不了的情況）
- 非 1st/2nd 連結，但對方是你非常想觸達的決策者

InMail 開頭不要廢話，直接說為什麼找他、有什麼具體價值。250 字以內。

### 用 TeamLink 找共同聯絡人

TeamLink 可以讓你看到你和對方之間有哪些共同聯絡人，這是比陌生觸達高 3 倍以上轉換率的方式。

**操作步驟：**

1. 在 Sales Navigator Lead 頁面，找到對方的 Profile
2. 右側面板找「TeamLink Connections」區塊
3. 如果有共同聯絡人，點進去看是誰

**有共同聯絡人時的應對：**

做法 A — 請對方幫你介紹（最有效）：
聯繫那位共同聯絡人，說：「我最近在幫 [對方公司] 評估 AI 工具整合，你跟王大明很熟對吧？方便幫我介紹一下嗎？」

做法 B — 在訊息裡提到共同聯絡人：
「您好，我是 [姓名] 的朋友介紹，他說您們公司在 AI 工具管理上可能有些需求...」

這個方法讓陌生觸達瞬間變成熟人介紹，信任感直接跳一個檔次。

### 追蹤 Lead 的 LinkedIn 動態（Job Change Alert）

**設定方式：**

2. 點「Save Lead」按鈕（如果還沒存）
3. 儲存後，系統會自動追蹤這個人的：
   - 工作職位變動

每週一打開 Alerts，你就會看到所有有動態的 Lead。

**Job Change 的處理邏輯：**

對方換新工作或升職 → 這是 Tier 2 訊號，今天內要行動：

如果這個人在你的 NURTURE 清單：
- 把狀態改回 WARM

- 把他加入今天的 HOT Lead 清單

### 用 Smart Links 追蹤對方是否點開資料

Smart Links 是 Sales Navigator 的功能，讓你把任何文件或連結包裝成一個追蹤 URL，可以看到對方有沒有點開、看了多久。

**適合傳什麼：**

**操作步驟：**

1. Sales Navigator → Smart Links → 「Create Smart Link」
2. 上傳檔案或貼入 URL
3. 建立後得到一個追蹤連結，複製這個連結傳給對方
4. 對方點開後，你的 Sales Navigator 首頁會出現通知

**Smart Link 通知的處理：**

對方點開了但沒回覆 → 這是個訊號，今天跟進：
「您好，不知道昨天分享的資料有沒有機會看到？如果有任何問題，歡迎直接問我。」

對方點開了又回覆 → 直接進 Demo 邀約流程。

**注意：** Smart Links 不要在每封訊息裡都放，放太多會讓對方覺得你在追蹤他。適合在 Touch 4 或 Touch 5 的時候用一次。

## Part 5：異常情境處理

這些情境不是每天都會碰到，但碰到的時候如果沒有 SOP 就會亂。先把這些讀一遍記在腦袋裡。

---

### 情境 1：LinkedIn 帳號被限制（每日連結上限觸發）

**什麼情況會發生：**

你一天發了超過 15 則連結請求，或者短時間內大量發訊息，LinkedIn 會限制你的帳號，顯示「You've reached the weekly invitation limit」或類似訊息。

**立刻要做的 3 件事：**

**步驟 1 — 停止所有連結請求，今天不再發（第 0 分鐘）：**
不管是不是快要達成今天目標，今天的連結請求全部停掉。繼續硬發只會讓限制更嚴重。

**步驟 2 — 轉換模式：用訊息替代連結請求（第 0~10 分鐘）：**
今天剩下的時間，改成處理已連結的 Lead 的訊息，不要觸碰連結請求功能。

**步驟 3 — 明天恢復節奏，每天不超過 10 則（未來幾天）：**
帳號解除限制後，頭三天每天只發 10 則（不是 15 則），讓系統認為你行為正常，再慢慢回到 15 則。

**預防措施：**
- 每天的連結請求最多 15 則，這是鐵則
- 不要在一個小時內連發超過 5 則
- 早上、中午、下午各分 3 批發送，分散時間點

在 Asana 記錄這次限制發生的日期和大概原因，讓自己以後知道要注意。

---

### 情境 2：對方封鎖你

**怎麼知道對方封鎖你：**
- 你的訊息無法送達（顯示錯誤）
- 對方的 Profile 你找不到了
- 對方從你的連結列表消失

**在 Asana 的記錄方式：**

1. 找到這筆 Lead 的 Task
2. 狀態改為 CLOSED-LOST
3. 在備注欄位寫：

```
2026-05-30 | 對方封鎖
可能原因：[你的判斷，比如「Touch 頻率太高」「訊息銷售感太重」]
學習點：[具體的改進方向]
```

4. 在 Task 的自定義欄位「結案原因」選「對方封鎖」

**後續：**

不要嘗試用其他帳號再次觸達同一個人，這樣做不只是浪費時間，也會傷害公司品牌。

每個月月底回顧被封鎖的 Lead，找出共同點（是不是同一個行業、同一個 Touch 節點），告訴 Kid 和 Alice，調整觸達策略。

---

### 情境 3：發現同一家公司有多個 Lead

你在 Asana 裡發現同一家公司有兩個或三個 Lead，都在不同的觸達階段。這個情況要協調，不然可能同一家公司收到好幾封訊息，顯得很沒組織。

**處理步驟：**

**步驟 1 — 確認關係：**

在 Asana 找出同一家公司的所有 Task，搞清楚：
- 誰是最終決策者？（通常是職稱最高的那個）
- 誰是影響者？
- 誰是執行層？

**步驟 2 — 決定主觸達窗口：**

原則：一家公司，同一時間只主動觸達一個人（最高決策者優先）。

另外幾個 Lead 的 Task 狀態改為 NURTURE，備注「同公司 [主要聯絡人姓名] 進行中，等主線有進展再處理」。

**步驟 3 — 等主線有了結果，再決定是否擴展：**

| 情境 | 下一步 |
|-----|-------|
| 主要決策者說「可以，我讓 IT 主管跟進」| 主動告知 IT 主管，你們在跟他們老闆談，現在要邀他一起了解 |
| 主要決策者說不需要 | 其他 Lead 也改為 CLOSED-LOST，這家公司暫時結案 |
| 主要決策者沒有回應 | 45 天後考慮改觸達影響者，但要換一個話題切入 |

---

### 情境 4：收到 Lead 主動詢問價格（1 小時內回覆流程）

這是最好的事情，也是最不能搞砸的事情。對方主動問你價格，代表購買意願分數直接 5 分。

**1 小時內必須完成的 5 個動作：**

**動作 1（0~5 分鐘）— 在 Asana 更新狀態為 LEAD-HOT：**

找到這筆 Lead 的 Task，購買意願分數改為 5，狀態改為 LEAD-HOT，備注對方的原始訊息。

**動作 2（5~10 分鐘）— 傳訊給對方，先問一個問題，不要直接報價：**

```
[對方名字] 您好，很高興您有興趣！

在我給您完整的方案建議之前，想先了解一下您們的使用情況，
這樣我給的報價才會最精準，不會讓您付了不需要的功能的錢。

請問您們團隊大概有幾個人在使用 AI 工具？
（包括工程師、行銷、業務都算進去）
```

為什麼不直接報價：因為 AI Token King 的定價是按用量和人數的，你不知道對方的規模，給出去的數字可能嚇跑對方，或者低估了你的機會。

**動作 3（10~15 分鐘）— 通知 Alice 準備報價單：**

在 Asana 或 LINE 告訴 Alice：「有個 Lead 問價格，公司 [名稱]，行業 [行業]，等對方回答人數再請妳準備報價。預計今天或明天要用。」

**動作 4（對方回答後，約 10~30 分鐘後）— 跟 Alice 確認報價：**

Alice 根據人數和使用場景，準備好報價單。你確認金額和方案組合，確保符合對方的情況。

金額授權確認：
- ≤ NT$50,000 → Kid + Alice 確認即可
- NT$50,001 ~ NT$200,000 → 需要 Lauren 連署
- > NT$200,000 → 需要 Jet 簽核

**動作 5（拿到報價後）— 邀約 Demo 而不是直接寄報價單：**

```
[對方名字]，根據您說的 [人數] 人的規模，我們有一個最合適的方案。

我建議這樣：我們先約 20 分鐘，我用畫面讓您直接看到平台操作，
同時我把方案細節和定價在會議中說明給您聽，您也可以當場問問題。

這樣比我用訊息說清楚快很多，您今天或明天哪個時段方便？
```

不要直接用訊息傳報價 PDF，Demo 轉換率遠高於文字報價。
