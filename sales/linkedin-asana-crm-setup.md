### 1.2 設計哲學：Pipeline as Truth

**核心原則：Asana 是唯一的真相來源（Single Source of Truth）。**

所有 Lead 的狀態、訊息紀錄、評分結果、Agent 輸出，全部集中在 Asana。任何口頭承諾、LINE 訊息、電子郵件討論，若未更新至 Asana，視為不存在。

```
行動守則：
1. 沒有 Asana 卡片 = 沒有這個 Lead
2. 沒有更新欄位 = 沒有發生過
3. Agent 輸出必須寫入對應 Custom Field，才算完成任務
4. Kid 每日結束工作前，所有進行中卡片必須更新「下次觸達日期」
```

### 1.3 與 linkedin-crm-tracking-flow.md 的對照關係

`linkedin-crm-tracking-flow.md` 定義的是**流程邏輯**（什麼情況做什麼判斷），本文件定義的是**系統設定**（在 Asana 裡如何建置這些判斷的載體）。

| 追蹤流程文件（HOW TO DECIDE） | 本文件（HOW TO BUILD IN ASANA）|
|-------------------------------|-------------------------------|
| 狀態標籤（LEAD-NEW 等 15 種）| 對應 Section 名稱與移動規則 |
| Gate 0 ~ Gate 4 核查清單 | 對應 Subtask 勾選項目 |
| HOT/WARM/COLD 分類決策樹 | 對應 Custom Field「HOT/WARM/COLD 分類」|
| 觸達序列 D1/D3/D7/D14/D21/D30/D45 | 對應 Custom Field「當前 Touch 編號」+ Automation Rule |
| 購買意願評分（1-5 分）| 對應 Custom Field「購買意願分數」|
| 升報機制（Kid → Lauren → Jet）| 對應 Automation Rule 第 10 條 |

## Part 4: Task 命名規範與子任務自動生成規則

### 4.1 Task 命名格式

**標準格式：**
```
[{分類}-{ICP分數}] {姓名} | {職稱} | {公司名稱} | {分數}分
```

**範例：**
```
[HOT-85] 王小明 | IT 主管 | 睿智科技 | 85分
[WARM-62] 陳美玲 | 行銷總監 | 數位創新股份有限公司 | 62分
[COLD-41] 林建國 | CEO | 微型新創 | 41分
[HOT-91] David Chen | CTO | CloudFirst Solutions | 91分
```

**命名規則說明：**

| 欄位 | 規則 |
|------|------|
| `{分類}` | 對應 B8 欄位：HOT / WARM / COLD |
| `{ICP分數}` | 對應 B1 欄位，取整數 |
| `{姓名}` | 中文全名或英文名（依 LinkedIn 顯示）|
| `{職稱}` | 精簡職稱（最多 6 個中文字或 20 個英文字元）|
| `{公司名稱}` | 若超過 10 字可縮寫，但需在 C10 備註欄寫完整名稱 |
| 分類變動 | 當 HOT/WARM/COLD 分類改變時，Kid 必須手動更新 Task 名稱前綴 |

## Part 9: Scout 評分模型的數據回饋迴圈

### 9.1 六個回饋維度完整定義

**維度 1：成交 Lead vs 未成交 Lead 的 ICP 分數分布差異**

```
分析方法：
  每月從 Section 08（成交）和 Section 09（結案）各取所有 Lead 的 B1 ICP 總分
  計算兩組的平均值、中位數、25/75 百分位數
  若成交組平均分 < 未成交組 15 分以上 → 評分模型識別力下降，需重新校正

關注指標：
  成交組 ICP 均值目標：≥ 75 分
  未成交組 ICP 均值預期：≤ 60 分
  兩組分數重疊比例目標：< 20%（重疊越少，模型區分力越強）

Asana 欄位數據來源：B1 群組所有欄位 + Section 最終狀態
```

**維度 2：HOT/WARM/COLD 分類準確率（回顧性分析）**

```
定義「準確」：
  HOT（≥70 分）Lead 最終成交 → True Positive
  COLD（<50 分）Lead 最終結案 → True Negative
  HOT Lead 結案 → False Positive（評分過高）
  COLD Lead 成交 → False Negative（評分過低）

目標準確率：
  HOT → 成交轉化率 ≥ 25%
  WARM → 成交轉化率 ≥ 10%
  COLD → 成交轉化率 ≤ 5%

異常觸發：若 HOT 成交轉化率 < 15%，觸發 Part 8 異常 02
```

**維度 3：哪些行業 + 職稱組合成交率最高**

```
交叉分析表格（每季更新）：
```
