## 前言：Frank 哲學的核心轉譯

在開始任何一套策略之前，必須內化一件事：

**你不是在賣軟體，你是在成為某個人發展路徑上的必要夥伴。**

這份 SOP 的所有話術，都建立在一個假設上——對方在你接觸之前，從未感受過「有人真的研究過我」。大多數業務的訊息是：「您好，我們有 AI 工具，想跟您介紹一下。」你的訊息要讓他感覺：「這個人在說的，是我最近一直在想的事。」

這個差距，就是你的勝負手。

## 模組零：AI 使用者類型分類系統

在情報蒐集之後，你必須先完成這個分類，才能進入話術選擇。

### 四大類型定義

---

**Type A — 身份建構者（Identity Builder）**

- **特徵**：履歷有明顯的職涯轉型，例如傳統業務跳到數位行銷、製造業出身後讀 MBA、非技術背景但在 LinkedIn 大量分享 AI 相關文章。
- **他的發展邏輯**：他正在重新定義自己是誰，AI 對他來說是「升級自己標籤」的工具，不是效率工具。
- **他想成為的 Position**：那個在舊行業裡最懂新科技的人，或者那個從傳統背景出發、成功完成數位轉型的人。
- **他期待被誰需要**：被同樣在轉型路上掙扎的人仰望，被新一代視為可以帶路的前輩。
- **核心恐懼**：被時代淘汰，或者轉型失敗後兩頭空。
- **AI Token King 對他的意義**：讓他有具體的「轉型裝備」，而且是他能對外展示的那種。

---

**Type B — 效率優化者（Efficiency Maximizer）**

- **特徵**：職稱通常是 Operations、PM、業務主管、HR 主管。LinkedIn 貼文偏向流程優化、工具比較、效率方法論。履歷顯示他在同一家公司待了很長時間，或者每換一份工作都是往上的跳法。
- **他的發展邏輯**：他是靠把事情做好、做快、做穩來建立價值的。他對 AI 的態度是：「能幫我省時間、省人力就好，別讓我踩雷。」
- **他想成為的 Position**：那個讓公司運作最順的關鍵人，不一定是最前台的，但是缺了他什麼都動不了。
- **他期待被誰需要**：被老闆信任的執行者，被團隊依賴的問題解決者。
- **核心恐懼**：引進一個工具之後出了問題，讓他在老闆面前失分。
- **AI Token King 對他的意義**：可靠的、有人顧的、出了事有窗口的那種服務，不是他需要自己搞懂的技術產品。

---

**Type C — 影響力建構者（Influence Builder）**

- **特徵**：有在 LinkedIn 持續寫文章、有個人品牌意識、追蹤數相對高、職稱常見 Founder / Co-Founder / 顧問 / 講師 / 獨立創業者。
- **他的發展邏輯**：他在建立一個以自己為核心的影響力資產。AI 對他來說是「讓我的觀點更快被更多人看到」的生產力擴張器。
- **他想成為的 Position**：某個領域的公認意見領袖（KOL），或者讓自己的品牌比公司的品牌更有價值。
- **他期待被誰需要**：被媒體邀稿、被活動邀請演講、被創業者詢問顧問合作。
- **核心恐懼**：掉隊——別人已經在用 AI 每天產出十篇文章，他還在手寫。
- **AI Token King 對他的意義**：讓他的產出速度和品質都超越同類型的競爭者，讓他的觀點傳播得更遠。

---

**Type D — 組織決策者（Organizational Decision Maker）**

- **特徵**：職稱 CEO / 總監 / VP / 事業部負責人。公司規模 50 人以上。履歷顯示他已經在管理層超過三年。LinkedIn 貼文少，但每次發都在談市場趨勢、公司方向、人才戰略。
- **他的發展邏輯**：他的 KPI 是整個組織的成果，他對 AI 的考量維度是：「這能讓我的團隊整體提升嗎？這會產生風險嗎？導入成本值得嗎？」
- **他想成為的 Position**：那個讓公司在這波 AI 轉型中沒有掉隊的決策者，或者讓公司因為他的決策而超越競爭對手的人。
- **他期待被誰需要**：被董事會 / 股東 / 投資人認可為有遠見的領導者，被員工視為帶領公司走對方向的舵手。
- **核心恐懼**：做了錯誤的工具選擇，導致資源浪費、或者讓公司在轉型上落後對手。
- **AI Token King 對他的意義**：降低組織導入 AI 的決策風險，並且有清楚的 ROI 說明讓他可以對上對下都交代得清楚。

## 情報蒐集通用框架（四套策略共用）

在進入任何一套策略的話術之前，必須完成這份情報清單。

### LinkedIn 蒐集欄位

```
個人資料層：
  - 現職職稱 + 公司名稱
  - 過去三份工作的職稱與年資
  - 學歷背景（科系、是否有在職進修）
  - 自我介紹（About 欄位逐字讀完）
  - Skills 欄位（排列前三的技能是什麼？）
  - 近三個月的貼文（主題是什麼？語氣是什麼？）
  - 按讚或留言過的內容（可以判斷他真正在意什麼）
  - 推薦信（別人怎麼說他的？）

公司資料層：
  - 公司 LinkedIn 主頁的近期動態
  - 公司員工人數變化（是在擴張還是縮減？）
  - 公司官網的最新新聞 / 媒體報導
  - 公司是否最近有融資、得獎、異動消息
```

### 104 蒐集欄位（S2 策略專用）

```
職缺層：
  - 職缺名稱（有沒有 AI / 數位 / 自動化相關關鍵字）
  - 職缺描述（要求技能中是否出現 ChatGPT / AI工具 / 流程自動化）
  - 刊登時間（越近代表需求越急迫）
  - 同一家公司同時刊登幾個相關職缺（越多代表轉型需求越強）

公司層：
  - 員工規模
  - 主要業務描述
  - 福利待遇（可以判斷公司財力與文化）
```

### 發展邏輯判斷三步驟

```
Step 1 — 描繪他的路徑：
  「他從什麼背景出發？他的每一次換工作是在往哪個方向走？」
  → 是往管理層走？往技術深度走？往創業方向走？往影響力方向走？

Step 2 — 找到他的轉折點：
  「他的履歷中有沒有一個明顯的跳躍？那個跳躍代表什麼決定？」
  → 這個轉折點通常是他最在意的身份轉變，也是你最能共情的切入口。

Step 3 — 定位他的下一步：
  「他現在的狀態，是在為下一個什麼目標鋪路？」
  → 這個答案決定你在他心目中要扮演什麼角色。
```

## S1 直攻法

### 策略定義

主動搜尋符合 ICP 的目標人物，發出連線邀請，建立初步信任後切入開發。

適用於：你已經有清楚的目標行業或職稱，願意用量換質，每週穩定開發 20 個以上新目標。

### 情報蒐集重點（S1 特定）

S1 的情報蒐集必須在發連線之前完成，這是最容易被跳過、也最不能跳過的步驟。你花在研究上的 10 分鐘，會讓你的連線接受率從 20% 提升到 60% 以上。

重點看：他的 About 欄位第一段（他如何定義自己？）、最近兩篇貼文（他在關心什麼話題？）、他的職稱轉變（他是向上走、向外走還是向深走？）。

---

### 話術設計

**連線邀請文（150字以內）**

```
[對方名字]你好，

看到你在 [公司名] 負責 [職能方向]，
剛才讀了你分享的那篇關於 [具體貼文主題或行業觀點]，
你說的「[引用他說過的一句話或一個觀點]」這個點
我覺得說到了很多人沒想清楚的地方。

我在兌心科技做 AI 工具的落地，
想跟你多交流——加個連線，保持聯繫。

Kid
```

**心理狀態說明：** 你在這一刻不是業務，你是一個認真讀了他文章的人。連線邀請的目的是讓他感覺「這個人有來過我的主頁，不是機器人群發」。你的心態要是：我是真的對他有好奇心，我想認識他這個人。

---

**第一封私訊（連線接受後 1-2 天，不推銷）**

```
[對方名字] 謝謝你接受連線。

我最近在觀察 [他所在行業] 這塊，
發現很多公司在 AI 工具這件事上卡在同一個地方——
不是不想用，是不知道從哪裡開始，用了也不知道有沒有效。

想請教你一個問題：
以你現在的工作情境，你覺得 AI 工具目前對你來說
最有感的是哪一塊？還是說還沒找到真正好用的？

純粹好奇，沒有要推銷任何東西。
```

**心理狀態說明：** 這封信的目的是讓他開口。只要他回覆，不管說什麼，你們就建立了對話關係。你的心態要是：我在做一個真誠的市場研究，他的回答對我真的有價值。不要急著講產品，急了就輸了。

對方心理：他剛接受連線，防備心還在。這個問題對他來說是低風險的——因為你說了「不推銷」，而且問的是他的經驗，人對分享自己的經驗是有天然意願的。

---

**第二封跟進（收到回覆後，或第一封發出後 5-7 天無回覆時）**

有回覆版本：

```
謝謝你的分享，你說的 [複述他說的痛點或觀點] 這個點，
我在跟其他 [同行業] 的朋友交流時也常聽到類似的狀況。

我最近整理了一些 [他的行業] 在 AI 應用上
真的跑出成果的案例，不是廠商的業配案例，
是實際從業者分享的實踐方式。

如果你有興趣，我可以傳給你參考——
不是要約你買東西，就是覺得你應該看得上。
```

無回覆追蹤版本：

```
[對方名字] 再打擾一下，

上次問了一個問題，可能時機不對或訊息被漏掉了。

我最近剛好在整理一份
「[他的行業] × AI 實際落地」的觀察整理，
想說如果你有興趣，傳給你看看。

不用回我沒關係，
只是覺得跟你的工作方向可能有一些交集。
```

**心理狀態說明：** 這封信的目的是製造一個他容易說「好」的低門檻動作。你給的是「資訊」不是「報價」，這讓他接受起來沒有心理負擔。你的心態要是：我是真的有東西可以給他，不是在找藉口聊天。

---

**第三封（引入方案，以「成為他所需角色」為框架）**

```
[對方名字]，

這段時間跟你的交流，我有一個觀察想直接說：

你在 [他的行業/職能] 這個位置，
接下來如果要繼續往 [他發展邏輯中的下一步] 走，
有一件事會是關鍵——就是你的 AI 工具的應用方式，
能不能真的跟你的工作邏輯對接，而不是你去配合工具。

我們在做的 AI Token King，
設計的出發點就是這個：不是給你一堆功能讓你自己摸，
而是根據你的工作情境，讓你的 AI 使用方式跑起來。

如果你願意給我 20 分鐘，
我不是要做簡報給你看，
是想直接讓你操作一次，感受一下跟你現在的狀況有沒有對接。

約個時間？
```

**心理狀態說明：** 你在這封信裡的角色，是「他路徑上的資源夥伴」，不是銷售。你的切入點是他的發展邏輯，不是你的產品功能。開口前先確認你真的讀懂了他是誰、他想去哪裡，不然這封信會變成普通的業務推銷。

## S2 104 意圖前置法

### 策略定義

從 104 職缺刊登中識別有 AI 採購意圖的公司，鎖定 LinkedIn 上的決策者，在競品接觸之前完成第一次觸達。

這是訊號驅動開發的核心體現。一個公司在 104 上刊登「需要熟悉 AI 工具的業務人員」，代表這家公司的決策者已經在考慮 AI 導入，但還沒有完成採購決策。這個窗口通常只有 2-4 週。

### 情報蒐集重點（S2 特定）

```
Step 1 — 104 訊號識別：
  搜尋關鍵字組合：
  → 「AI工具」「ChatGPT」「Copilot」「自動化」「流程優化」+「業務」「行銷」「客服」「營運」

  高意圖訊號（立刻觸達）：
  → 職缺描述中明確提到「需要使用AI工具提升效率」
  → 同時刊登 3 個以上相關職缺（規模性導入需求）
  → 職缺刊登時間在 14 天以內

  中意圖訊號（本週觸達）：
  → 職缺提到「數位化轉型」「工作流程現代化」
  → 公司同時在招 IT 或系統管理職位

Step 2 — 決策者定位：
  在 LinkedIn 搜尋該公司，找以下職稱：
  → CEO / 總經理 / 執行長（50人以下公司首選）
  → 業務總監 / 行銷總監 / 營運總監（對應職缺所屬部門）
  → IT 主管 / 數位轉型負責人

Step 3 — 決策者背景研究：
  對照通用框架完成情報蒐集，
  特別注意：他在這家公司的任期長短（越新的主管越有採購動機）
```

### 話術設計

**連線邀請文（150字以內）**

```
[對方名字] 你好，

我剛在 104 上看到 [公司名] 在招 [職缺名稱]，
裡面提到了 [具體的AI工具相關描述]，
感覺你們正在認真推進這個方向。

我是 Frank，AI Token King 的創辦人，
我們在幫台灣 AI 團隊解決多工具、多模型的用量管理問題。
想跟你連線保持聯繫——說不定有交流空間。

Frank
```

**心理狀態說明：** 你讓他知道你「不是亂打」——你有具體的觸發點（104 職缺），這代表你是有備而來的。你的心態是：我是在這個行業深耕的人，我看到了一個交集，我想確認一下。Frank 以創辦人身份出現，對 CTO / 主管層級是對等的對話，對工程師則是「產品建立者」的真實感。

**第一封私訊**

```
[對方名字] 謝謝連線。

看到你們最近在招 [職缺名稱]，
我做了一個合理的推測——
[公司名] 可能正在考慮怎麼讓 AI 工具
真正整合進工作流程，而不只是讓員工「自己去用」。

如果我猜對了方向，我可以分享一些
我們最近陪幾家 [同行業] 公司走這個過程的觀察——
包括哪些地方踩雷了，哪些做法跑出效果了。

你方便的話，我把整理好的內容傳給你看看？
```

**心理狀態說明：** 你用「合理推測」而不是「我知道你的需求」，這個措辭很重要——它讓他感覺你是在猜想，而不是在宣稱你了解他，不會讓他產生被監視的感覺。你的心態是：我可能是他現在需要的那個人，我要先確認。

---

**第二封跟進**

```
[對方名字]，

多想了一下你們的情況。

你們現在招的這個職位，
如果是在考慮讓員工用 AI 工具提升產出，
通常最快碰壁的地方會是：
工具很多，但沒有一個統一的使用標準，
結果變成每個人自己摸索，效率反而參差不齊。

我們剛好有一個做法是幫企業建立「AI工具使用的標準作業框架」，
讓整個團隊的工具用法一致、結果可以被追蹤。

你們有沒有在想這個層面的問題？
還是你們目前規劃的方向跟我想的不同？
```

**心理狀態說明：** 這封信開始展示你的專業洞察，讓他感受到「這個人比我更早想到這個問題」。你不是在回答他的問題，你是在引導他發現他還沒意識到的問題。這是建立專業信任的核心動作。

---

**第三封（引入方案）**

```
[對方名字]，

我想直接說：

你們現在招這個職位的時間點，
是一個很好的機會——因為這代表公司已經準備好在 AI 工具這件事上做一些改變。

但如果工具選型沒有在這個時間點想清楚，
等新人進來之後再來改，成本會高很多。

AI Token King 我們的設計就是為了這個時間點：
讓你在人員到位之前，先把 AI 工具的框架搭好，
新人進來就可以在對的軌道上用對的工具。

我不希望你在採購之後才來找我——
希望我能在你們還在規劃的時候就介入，
幫你把這個決策做得更紮實。

給我 20 分鐘？我直接帶你看一遍。
```

**心理狀態說明：** 你在這裡的角色是「不讓他踩雷的那個人」，而不是「賣東西的那個人」。你的緊迫感來自他的利益（決策時間點），而不是你的業績壓力。這個差別對方是感受得到的。

## S5 社群潛伏法

### 策略定義

加入目標客群聚集的 LinkedIn 社群，長期建立可見性與專業形象，讓目標對象主動對你產生好奇心，降低冷觸達的阻力。

這是四套策略中週期最長、但信任建立最深的一套。適合你有時間經營、想在特定行業建立長期影響力的情況。

### 情報蒐集重點（S5 特定）

```
社群研究：
  - 找到 3-5 個目標行業的活躍 LinkedIn 社群或 hashtag
  - 觀察裡面討論最熱烈的主題是什麼
  - 找出哪些人是高互動的意見領袖（不一定是追蹤數最高的）

目標人物研究：
  - 在社群中找到符合 ICP 的人
  - 觀察他們在社群裡的發言風格（提問型 / 分享型 / 辯論型）
  - 了解他們最在意的議題是什麼

你自己的定位研究：
  - 在這個社群裡，你要扮演什麼角色？
  - 你的貼文主題要如何讓目標受眾感覺「這個人是懂我的行業的」？
```

### 話術設計

**連線邀請文（建立在互動之後）**

```
[對方名字] 你好，

上週看到你在 [社群名稱] 留言說的那一段——
「[引用他說的話]」，

我覺得你點出了一個很多人繞開不說的問題。
想跟你多交流，加個連線。

Kid
```

**心理狀態說明：** 因為你已經在社群裡互動過，這個連線不是冷觸達，而是「延續一段已經存在的對話」。你的心態是：我認識你了，我想認識得更深一點。這封邀請的重點是讓他想起你是誰。

---

**第一封私訊**

```
[對方名字]，

我在 [社群名稱] 觀察了一段時間，
發現你對 [他常討論的議題] 有很持續的關注。

我有一個問題一直想問：
你認為 [他的行業] 在 AI 工具這件事上，
目前最大的阻力是工具本身的問題，還是使用者的問題？

我問這個是因為我在做這個領域的事，
很想聽真正在業界的人的直覺判斷——
不是廠商視角，是使用者視角。
```

**心理狀態說明：** 你用一個有深度的問題邀請他進入對話。這個問題沒有標準答案，而且它預設了你對他的行業是有基本認識的。你的心態是：我是在做研究的人，他的觀點對我有真實的價值。

---

**第二封跟進**

```
[對方名字]，

你上次說的「[複述他的觀點]」，
我跟幾個同行交流後，
發現這個觀點跟實際跑出來的結果是高度吻合的。

我整理了一些具體的案例，
就是在你說的那個阻力點上，
有幾個團隊是怎麼突破的——

想傳給你看，不是廠商案例，
是我們從實際使用中觀察到的模式。
你有興趣嗎？
```

**心理狀態說明：** 你用行動讓他感受到「他的話語被認真對待了」。這是建立信任最快的方式——不是你說你重視他，是你用行動證明你在聽他說話。

---

**第三封（引入方案）**

```
[對方名字]，

跟你交流這段時間，
我感覺你對 AI 工具這件事的判斷已經很清晰了——
你知道什麼有用、什麼是噪音。

正因為這樣，我才想直接問你：

你目前的工作情境，
有沒有哪一塊是你覺得「如果有對的工具配合，

```

**心理狀態說明：** 這封信的最後一段是「恭維性的邀請」——你告訴他他是那種值得用好工具的人。這比說「我們的產品很好」更有力量，因為你在說的是他的能力，不是你的產品。你的心態是：我在邀請一個我欣賞的人嘗試一件事。

## S6 競品替換法

### 策略定義

識別正在使用競品工具的目標對象，針對競品的已知痛點精準切入，在對方考慮換約或累積足夠不滿的時刻觸達。

這是四套策略中精準度最高、但準備工作最重的一套。你必須真的了解競品的弱點，才能執行這套策略，否則話術會顯得空洞。

### 情報蒐集重點（S6 特定）

```
競品痛點研究：
  - 查看競品的 G2 / Capterra 評價（負評集中在哪裡？）
  - 查看競品的 PTT / Dcard / 行業論壇討論
  - 詢問已換過來的客戶：他們為什麼離開競品？

目標對象識別：
  - LinkedIn 個人技能欄位出現競品名稱
  - LinkedIn 貼文提到正在使用某競品
  - 104 職缺描述要求「熟悉 [競品名稱]」
  - 公司官網案例提到使用競品

替換時機訊號（高優先）：
  - 競品最近有漲價、功能刪除、服務異動消息
  - 目標公司最近有人事異動（新主管 = 新評估機會）
  - 目標對象最近在 LinkedIn 發文抱怨工具問題
```

### 話術設計

**連線邀請文（150字以內）**

```
[對方名字] 你好，

看到你在用 [競品名稱] 做 [功能應用]，
我在這個領域做了幾年，
對這類工具在 [目標行業] 的實際落地有些觀察。

想跟你連線——
不是要說競品不好，
是覺得你對這塊的實際使用經驗
應該有很多我沒有想到的角度。

Kid
```

**心理狀態說明：** 這封信刻意說「不是要說競品不好」，這是一個重要的心理安全感建立——你讓他知道你不是來攻擊他的選擇，你尊重他已經做的決定。這讓他的防備心降低，更願意進行真實的交流。

---

**第一封私訊**

```
[對方名字]，

謝謝你接受連線。

我想直接問你一個真實的問題：
你目前用 [競品名稱] 這段時間，
有沒有哪些地方是你還沒找到最好解法的？

我問這個不是要推銷，
是因為我在陪幾家同類型公司做 AI 工具評估，
他們在 [競品常見痛點，例如：價格結構/中文支援/整合彈性] 上
都有一些共同的卡點。

不確定你們有沒有類似的狀況，
想直接問你比較快。
```

**心理狀態說明：** 你在這裡做的事是「精準預測他的痛點」——你提到的競品常見問題，如果他有類似的感受，他會立刻感覺「這個人懂」。這是建立共頻最快的方法。你的心態是：我不是來否定他的選擇，我是來看看有沒有他還沒解決的問題。

---

**第二封跟進**

```
[對方名字]，

你上次說的 [他提到的痛點]，
我其實一點都不意外——

[競品名稱] 在這個點上的設計邏輯，
是針對 [競品主要市場] 的使用情境，
在 [目標行業] 的實際場景裡，
那個設計反而會產生阻力。

我們在 AI Token King 的設計，
有一個很不一樣的出發點——
是先從 [目標行業] 的工作邏輯反推，
再決定功能怎麼做。

不是要說我們比較好，
是我覺得你們的狀況值得讓你看看這個差別。
```

**心理狀態說明：** 這封信要展示「你對競品的理解是深的」——你在解釋為什麼競品在他的情境裡會有問題，而不只是說「我們比他們好」。這個差別代表你有專業度，而不只是在攻擊競品。

---

**第三封（引入方案）**

```
[對方名字]，

我想說一件直接的事：

大部分人換工具，
是因為痛到受不了才換，
然後換的過程又要花很多時間和學習成本，
所以很多人明明知道現在的工具有問題，
還是選擇繼續用。

我不希望你是這個情況。

我想做的事是：
先讓你花 20 分鐘，
看看 AI Token King 跟你們現在的工作方式
到底有多少磨合空間——

如果磨合成本大於效益，我會直接告訴你。
如果值得換，你自己看了會知道。

你願意給我這 20 分鐘嗎？
```

**心理狀態說明：** 你在這裡做了一件很重要的事——你說「如果不值得換，我會告訴你」。這是建立信任最快的動作：你的建議不是為了你自己的利益，而是為了他的利益。這讓他感覺你不是業務，是顧問。

## 通用心理防線破解指引

在所有策略的執行過程中，你會遇到以下四種常見阻力：

---

**阻力一：「我現在很忙，沒有時間看這個」**

不要解釋你只需要幾分鐘。

正確回應：「完全理解，等你有空的時候再說。我先把一個東西傳給你，不需要回我，放在那裡，哪天有感覺了再看。」

心理邏輯：你讓他感覺沒有壓力，反而提升了他之後回來的意願。

---

**阻力二：「我們有 IT 部門在評估，不是我決定」**

不要說「那我找 IT 部門」。

正確回應：「了解，那我想問你一個問題：如果 IT 那邊給了你選項，你在意的點是什麼？因為最後用的人是你的團隊，你的角度很重要。」

心理邏輯：你把他重新變成有意見的人，而不是被排除在決策外的人。

---

**阻力三：「我們已經有在用其他工具了」**

不要說「我們比他們好」。

正確回應：「好，我很好奇——你們用了多久了？目前跑下來，你覺得最滿意的是哪一塊？」

心理邏輯：先讓他說工具的好，他說完之後，你問：「那有沒有哪一塊，還沒找到最好的解法？」他的回答會告訴你切入點在哪裡。

---

**阻力四：「價格怎麼算？」（太早問價格）**

這個問題在你還沒建立清楚的價值感之前出現，代表他在用價格當篩選器，而不是在認真評估。

正確回應：「價格這個我一定會讓你清楚，但我想先讓你看到你的情境適合用哪種方式——因為不同的導入規模，費用結構差很多。可以先給我 15 分鐘了解一下你的狀況嗎？」

心理邏輯：你不是在逃避報價，你是在說「我要讓你的錢花得值得」，這是站在他的利益說話。

## 最後一件事：業務的心理武裝

在你發出每一封訊息之前，問自己這三個問題：

**第一，如果我不是在賣東西，我還會發這封信嗎？** 如果答案是否，重寫。

**第二，這封信說的，是他的事，還是我的事？** 如果超過一半在講你的產品，重寫。

**第三，如果他永遠不買，這段關係對他還有沒有價值？** 如果沒有，你建立的不是信任，是一個交易。

能做到這三件事的業務，不需要話術技巧——因為他本身就是最好的話術。
