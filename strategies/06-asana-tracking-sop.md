## 一、Lead 狀態定義

每筆 Asana 任務的 `備註（Notes）` 最後一行即為當前狀態，格式固定：

```
狀態: [狀態碼] / Kid / [日期]
```

### 狀態碼一覽

| 狀態碼 | 說明 | 下一步動作 |
|--------|------|----------|
| `待發連線邀請` | 尚未送出 Connection Request | 匯入 Dripify，啟動 Campaign |
| `連線邀請已發` | Connection Request 已送出，等待接受 | 等待接受，最多 21 天 |
| `連線已接受` | 對方接受連線 | Day 1 訊息由 Dripify 自動發送 |
| `Day1已發` | Day 1 訊息已送出 | 等待回覆；Day 7 跑 follow-up |
| `Day7已發` | Day 7 follow-up 已送出 | 等待回覆；Day 14 跑第三封 |
| `有回覆-待接球` | 對方已回覆，等 Kid 處理 | 24 小時內接球，判斷進入哪個路徑 |
| `接球中` | Kid 正在主動對話中 | 判斷是否符合繼續推進條件 |
| `約到通話` | 已排定 Discovery Call | 準備通話前簡報，填 Call Brief |
| `進入方案討論` | 對方表達興趣，進入報價/方案階段 | Alice 介入準備報價單 |
| `成交` | 合約簽署完成 | 交接 Yulenda Onboarding |
| `暫緩` | 時機未到，3 個月後重新接觸 | 設 Asana Due Date 為 3 個月後 |
| `不符合` | 確認非 ICP 或明確拒絕 | 關閉任務，備註原因 |
| `逾期-連線未接受` | 邀請送出後 21 天未接受 | 關閉任務或備用帳號重試 |

## 二、每日更新流程（Kid 執行）

### 早上（09:00 前）：昨日狀態同步
1. 打開 Dripify Dashboard → 查看昨日回覆與連線接受數
2. 對每個有動作的 Lead，進入對應 Asana 任務
3. 更新 Notes 最後一行的狀態碼
4. 如有回覆 → 狀態改為 `有回覆-待接球`，並在備註加一行：
   ```
   回覆摘要: [一句話描述對方說了什麼] / [日期]
   ```

### 下午（收盤前）：接球處理
1. 篩選所有 `有回覆-待接球` 的任務
2. 根據對方語氣/內容，選擇接球話術（參考 `05-talk-scripts.md`）
3. 回覆完成後更新狀態為 `接球中`

## 四、Dripify ↔ Asana 雙向對齊規則

| Dripify 事件 | Asana 動作 |
|-------------|------------|
| Connection Request 送出 | 狀態 → `連線邀請已發` |
| 對方接受連線 | 狀態 → `連線已接受` |
| Day 1 訊息自動送出 | 狀態 → `Day1已發` |
| Day 7 訊息自動送出 | 狀態 → `Day7已發` |
| 對方在 Dripify 有回覆偵測 | 狀態 → `有回覆-待接球`（Kid 手動確認）|
| 21 天無接受 | 狀態 → `逾期-連線未接受`，任務標記完成 |

> **注意：** Dripify 的自動偵測僅供參考，Kid 每天需手動進 LinkedIn 收件匣確認是否有回覆被遺漏。

## 五、升級處理標準

以下情況需在 Asana 任務中 Tag Lauren（@Lauren）：

| 情況 | 升級原因 |
|------|----------|
| 對方要求直接通話但 Kid 不確定如何定位方案 | 需要話術指導 |
| 對方是 C-Suite 且表達強烈興趣 | 可能需要 Lauren 或 Frank 親自出席 |
| 對方提到競爭對手或具體比較需求 | 需要 Alice 提供產品定位支援 |
| 對方直接詢問價格 | 轉交 Alice 準備報價單 |

## 六、每月回顧指標（月底）

| 指標 | 目標值（S1 第一個月）| 計算方式 |
|------|------------------|----------|
| 連線邀請接受率 | ≥ 35% | 接受人數 / 送出人數 |
| Day 1 回覆率 | ≥ 15% | 有回覆人數 / 連線已接受人數 |
| 約到通話率 | ≥ 5% | 通話數 / 連線已接受人數 |
| 平均接觸到通話天數 | ≤ 21 天 | 從連線接受到約到通話的間隔 |

月底由 Lauren 根據以上數據決定下一批 S1 名單的 ICP 條件是否調整。

## 七、任務命名規則（新增 Lead 時）

```
[優先級-TypeX-ICP] 姓名 | 職稱 | 公司名稱
```

範例：
```
[A級-TypeD-100] Ken Chan | CTO, VP | AIRA 城智科技
[B級-TypeA-70] Jane Lin | Marketing Manager | XYZ Corp
```

優先級定義：
- **A 級**：ICP ≥ 85，TypeD 或高潛力 TypeA
- **B 級**：ICP 70–84
- **C 級**：ICP < 70（備用名單，視資源決定是否啟動）
