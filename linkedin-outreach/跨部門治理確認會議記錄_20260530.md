## 二、訊息模板（三版）

### 版本 A：企業管理者（IT / 數位化主管）

> **切入角度：** 你們已經在用這些工具了，但這個問題你現在解不了

```
[姓名] 您好，

公司有人在用 ChatGPT、有人用 Gemini、有人用 Claude——
每月 AI 費用到底花了多少、花在哪，您現在看得到嗎？

我們把三個平台的用量和帳單整合在一起，
費用比分開訂閱便宜，還能設部門配額和用量預警。

可以給您一份各平台的費用比較試算表，您有興趣嗎？
```

> **給什麼：** 費用比較試算表（具體可交付的東西，不是「聊聊」）

---

### 版本 B：企業老闆 / 總經理

> **切入角度：** 你付錢，但不知道付了多少、付給誰

```
[姓名] 您好，

公司現在用的 ChatGPT、Claude 或 Gemini，
每個月 AI 花費您知道是多少嗎？

很多老闆到月底才發現帳單超標，或者某天服務突然斷了，
才知道 Token 早就用完了。

我們可以幫您把所有 AI 工具的費用集中管理，
比原價便宜，還能防止中斷。

我整理了一份「企業 AI 費用管控 SOP」，要不要先看看？
```

> **給什麼：** 企業 AI 費用管控 SOP（一頁簡報或 PDF，實用文件）

---

### 版本 C：代理商招募

> **切入角度：** 你的客戶正在付冤枉錢，你可以幫他們解決

```
[姓名] 您好，

您現有的客戶裡，有沒有在用 ChatGPT 或 Claude 的？

很多企業現在分開訂閱多個 AI 工具，費用高又難管理——
這個問題其實可以是您切入的服務機會。

我們提供代理商後台，讓您直接幫客戶統一管理 AI 費用、
設定配額、看分潤報表，完全自助，不需要人工介入。

我可以先把代理方案和毛利結構給您看，有興趣嗎？
```

> **給什麼：** 代理方案 + 毛利結構試算（讓對方看到自己能賺多少）

---

**三版訊息共同升級原則（v2）：**
1. **點名具體工具**：ChatGPT / Claude / Gemini，不說「AI 工具」這種模糊詞
2. **痛點要具體**：帳單超標、Token 中斷、費用不透明——讓對方自己對號入座
3. **每版都有一個「可交付的東西」**：試算表 / SOP / 毛利結構 → 對方回覆是為了拿到東西，不是被推銷
4. **結尾是選擇題，不是拜託**：「要不要先看看」「有興趣嗎」——主動但不卑微

## 三、治理細節客製化決議

### 3.1 發送配額（Q2 衝刺期）

| 類型 | 原設計 | AITokenKing 客製化 |
|------|--------|-------------------|
| Connection Request | ≤ 50/日 | **≤ 30/日**（品質優先） |
| InMail | ≤ 20/日 | **≤ 15/日**（代理商軌專用） |
| Follow-up | ≤ 30/日 | **≤ 20/日** |
| A 級 Lead 週上限 | 無限制 | **20 筆/週**（超過 hold 到次週） |

### 3.2 人工鎖定機制（Reserved List）

**更新（2026-05-30 補充）：** Kid 目前並未執行手動陌生開發，因此**無需維護 Kid 手上名單的黑名單**。所有陌生開發流程全部交由 Agent 執行，以「人啟動（Human-in-the-loop trigger）」模式運作——Frank 或 Jet 手動觸發 Orchestrator，其餘全自動。

- **Jet 指定的代理商對象**：Jet 可透過 Asana 標記 `reserved_jet`（自訂欄位），Agent 自動跳過
- 每週一 Orchestrator 同步最新黑名單，確保不重疊

### 3.3 Hot Lead 路由規則

```
回覆偵測 → Claude Haiku 分類職稱/意圖
  │
  ├── 「代理 / 通路 / 合作夥伴」相關
  │     → 推送 Slack #商務-Jet，附：公司背景摘要
  │
  ├── 「IT / 資訊 / 數位 / CIO / 主管 / 負責人」相關
  │     → 推送 Slack #CS-Yulenda，附：公司規模/AI工具摘要
  │     → Yulenda pre-qual 後手動轉 Kid
  │
  └── 模糊/其他
        → 推送 Slack #lead-review（人工判斷）
```

**禁止動作：** Track Agent 不得自動發送會議連結，一律由人工確認後手動邀約

### 3.4 Track Agent 自動補充查詢

每筆 Hot Lead 回覆後，Exa Search 自動查詢並附在通知卡片：
- 公司近 30 天是否有 AI 相關職缺（代表 AI 預算）
- 公司員工規模（估算 Token 用量）
- 是否使用 AWS AI / Azure AI / Google AI（競品偵測）
- 近期媒體報導或融資新聞
